Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The meta elements contain information about the page that is used by the browser.
  • The charset element sets our character encoding.
  • The http-equiv element sets an HTTP header for content.
  • The viewport element sets the default view for our webpage.
  • The div element is used to create a division in the page.
  • The span element is used to create an inline division.
  • The p element represents a paragraph or block of text.
  • The ul, ol and li elements represent unordered lists, ordered lists, and list items.
  • The img element contains information about images that are displayed on the webpage.
  • The br element creates a line of empty space, or a line break, between two blocks of content.
  • The a element (which stands for anchor) creates links to the same webpage or other webpages.
  • The href attribute points to the URL for the link.
  • The footer appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

CSS

  • When a developer wants to style an element directly in the HTML file, they use inline CSS.
  • The internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • External CSS style sheet allows developers to keep all their CSS rules in a separate file.
  • When using an external style sheet, we link the file using a link element placed in the HTML's head.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • To apply a rule to all the elements visible on the page we use *.
  • The display property is assigned with the block value, which assigns each image to start on a new line.
  • Class attributes allow us to share a CSS rule to any element we choose.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: Stage all new/edited files for commit
